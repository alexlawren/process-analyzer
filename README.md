# Анализатор процессов (Process Analyzer)

Консольная утилита для анализа системных процессов, разработанная в рамках лабораторной работы. Приложение является кроссплатформенным и предоставляет отчет об использовании ресурсов на основе правил, заданных в конфигурационном файле.

![CI Build](https://github.com/alexlawren/process-analyzer/actions/workflows/build.yml/badge.svg)

## 1. Функционал и технологии

### Основные возможности
*   **Сбор данных:** Автоматически выполняет системные команды (`tasklist` для Windows, `ps aux` для Linux) для получения списка активных процессов.
*   **Сортировка:** Сортирует процессы по убыванию использования CPU или оперативной памяти.
*   **Поиск:** Находит PID процесса по его имени.
*   **Фильтрация:** Отображает процессы, потребление ресурсов которыми превышает заданные пороги.

### Технологический стек
*   **Язык:** Java 17
*   **Система сборки:** Gradle
*   **Тестирование:** JUnit 5
*   **Статический анализ:** Checkstyle
*   **CI/CD:** GitHub Actions

## 2. Сборка и запуск

### Требования
*   Java Development Kit (JDK) версии 17 или выше.
*   Git для клонирования репозитория.

### Шаг 1: Клонирование репозитория
```bash
git clone https://github.com/alexlawren/process-analyzer.git
cd process-analyzer
Шаг 2: Конфигурация
Перед первым запуском необходимо настроить поведение приложения.
Создайте в корневой директории проекта файл app.properties.
Скопируйте в него следующее содержимое и отредактируйте под свои задачи:
code
Properties
# Режим сортировки: CPU или MEMORY
# Определяет, по какому параметру сортировать процессы в отчете.
# - CPU: по убыванию использования CPU (результат виден в основном на Linux).
# - MEMORY: по убыванию использования оперативной памяти.
sort.by=MEMORY

# Пороги для фильтрации процессов.
# Будут показаны процессы, которые превышают ХОТЯ БЫ ОДИН из порогов.
threshold.cpu.percent=10
threshold.memory.mb=1000

# Имя процесса для поиска.
# Будет найден первый процесс, имя которого содержит эту подстроку (без учета регистра).
search.process.name=java
Шаг 3: Сборка проекта
Проект использует Gradle Wrapper, поэтому установка Gradle не требуется.
Выполните команду в терминале из корневой папки проекта:
Windows:
code
Bash
gradlew build
Linux/macOS:
code
Bash
./gradlew build
Эта команда скомпилирует исходный код, запустит тесты и статический анализ, а затем упакует приложение в исполняемый JAR-файл. Готовый файл будет находиться по пути app/build/libs/app.jar (имя может незначительно отличаться).
Шаг 4: Запуск приложения
После успешной сборки и настройки app.properties, запустите приложение следующей командой:
code
Bash
java -jar app/build/libs/app.jar
В консоли вы увидите отчет, сформированный в соответствии с вашими настройками.
Пример вывода:
code
Code

3. Тестирование
Для проверки корректности бизнес-логики и стиля кода используется команда check. Она запускает как Unit-тесты, так и статический анализ Checkstyle.
code
Bash
./gradlew check
Ожидаемый результат — BUILD SUCCESSFUL. Детальные отчеты о тестировании и статическом анализе генерируются в директории app/build/reports/.
