\# Анализатор процессов (Process Analyzer)



\## 1. Краткое описание проекта



\*\*Process Analyzer\*\* — это кроссплатформенная консольная утилита для анализа системных процессов. Приложение получает список активных процессов, парсит его и предоставляет отчет на основе заданных в конфигурации правил.



\*\*Основное назначение:\*\* Быстрый анализ использования ресурсов (CPU, память) для выявления наиболее "тяжелых" процессов, а также поиск и фильтрация процессов по различным критериям.



\*\*Технологии:\*\*

\*   \*\*Язык:\*\* Java 17

\*   \*\*Система сборки:\*\* Gradle

\*   \*\*Тестирование:\*\* JUnit 5

\*   \*\*CI/CD:\*\* GitHub Actions



\## 2. Пример работы



Приложение не требует данных для стандартного ввода (stdin). Оно самостоятельно выполняет системные команды (`tasklist` для Windows, `ps aux` для Linux) и использует их вывод в качестве входных данных.

\## 3. Как выполнить сборку проекта



Для сборки проекта используется Gradle Wrapper, поэтому предварительная установка Gradle не требуется. Необходим только установленный JDK версии 17 или выше.



1\.  \*\*Клонируйте репозиторий:\*\*

&nbsp;   ```bash

&nbsp;   git clone https://github.com/alexlawren/process-analyzer.git

&nbsp;   cd process-analyzer

&nbsp;   ```



2\.  \*\*Выполните команду сборки:\*\*

&nbsp;   \*   Для Windows:

&nbsp;       ```bash

&nbsp;       gradlew build

&nbsp;       ```

&nbsp;   \*   Для Linux/macOS:

&nbsp;       ```bash

&nbsp;       ./gradlew build

&nbsp;       ```



Эта команда выполнит компиляцию кода, запустит все тесты и статический анализ, а затем упакует приложение. После успешной сборки исполняемый JAR-файл будет находиться в директории `app/build/libs/`.



\## 4. Как выполнить конфигурацию проекта



Все настройки приложения вынесены в файл `app.properties`. \*\*Перед первым запуском необходимо создать этот файл\*\* в корневой директории проекта (на одном уровне с `src`, `build.gradle`).



\*\*Содержимое и описание параметров `app.properties`:\*\*



```properties

\# Режим сортировки: CPU или MEMORY

\# Указывает, по какому параметру сортировать процессы в Задании 1.

\# - CPU: сортировка по убыванию использования CPU (актуально для Linux).

\# - MEMORY: сортировка по убыванию использования памяти.

sort.by=MEMORY



\# Пороги для фильтрации в Задании 3

\# Приложение найдет процессы, которые удовлетворяют ХОТЯ БЫ ОДНОМУ из условий.

threshold.cpu.percent=10

threshold.memory.mb=1000



\# Имя процесса для поиска PID в Задании 2

\# Приложение будет искать первый процесс, имя которого содержит указанную подстроку (без учета регистра).

search.process.name=java

Вы можете изменять эти параметры в соответствии с вашими задачами. Например, для поиска процессов Google Chrome можно установить search.process.name=chrome.

5\. Как запустить тесты

Тесты запускаются автоматически при сборке проекта (./gradlew build). Однако, если вы хотите запустить только тесты (включая статический анализ), используйте команду check.

Для Windows:

code

Bash

gradlew check

Для Linux/macOS:

code

Bash

./gradlew check

Ожидаемый результат:

После выполнения команды вы должны увидеть в консоли BUILD SUCCESSFUL. Это означает, что все Unit-тесты прошли успешно и код соответствует правилам Checkstyle. Детальный HTML-отчет о результатах тестов можно найти в app/build/reports/tests/test/index.html.
